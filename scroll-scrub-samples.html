How it works, in short

Each section above is one tall .scrub-section. Its left column is position: sticky; top: 0, so it locks in place for the full height of the right column's text. A single scroll listener computes progress = (scrollY − sectionTop) / (sectionHeight − viewportHeight), clamped 0–1, and every visual — node fills, typed characters, the countdown — is a direct function of that one number. Scroll up and it runs backward for free.
