Motion Patterns Lab

Thirteen ways to make the page do the talking.

A working sample of the patterns we discussed — sticky pins, horizontal tracks, parallax depth, scroll-scrubbed counters, and a few hover-driven extras. Set 2 (09–13) adds beam connectors, a self-driving chat demo, a bento grid with spotlight cards, a before/after slider and a logo marquee. Scroll down; every section is live.

01 · Scroll-scrubbed pin

The diagram stays put. The scroll drives which module lights up.

Progress through this section is mapped to a 0–1 value; each module crosses a threshold and lights up in turn. Scroll back up and it un-lights in reverse — nothing here is a one-shot trigger.

02 · Vertical scroll → horizontal track

Keep scrolling down; the cards move sideways.

Vertical scroll progress inside this section is read and mapped onto a horizontal transform. Common for galleries and step sequences where you want a filmstrip feel without a real horizontal scrollbar.

05 · Magnetic button

Hover near it — it leans toward the cursor.

Reads cursor position relative to the button's center and nudges it a few pixels, snapping back on mouse-leave.

06 · Cursor tilt

Cards rotate in 3D based on where the cursor sits.

Cursor position over each card maps to rotateX/rotateY, giving a light, physical response without a JS animation library.

09 · Beam connectors

The wires carry traffic.

Pulses travel module → core → AI → out, on repeat (SMIL animateMotion along the same paths the wires use — zero JS). Turns a static architecture diagram into a live system. Blue = internal flow, orange = leaving via MCP.

10 · Chat simulation

The demo asks itself the question.

The question types itself, the assistant "thinks", then the answer streams in with data bars. Loops while in view, resets when you scroll away. This is the MCP / ask-your-business pitch shown, not told.
